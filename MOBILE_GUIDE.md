# 모바일 실행 및 앱처럼 사용하기

## 1) 같은 와이파이에서 접속
1. PC에서 `run_server.bat` 실행
2. PC에서 `ipconfig` 입력 후 IPv4 주소 확인
3. 휴대폰 브라우저에서 `http://PC_IP:8000` 접속

예시: `http://192.168.0.15:8000`

## 2) 홈 화면에 추가
- 안드로이드 크롬: 메뉴 → 홈 화면에 추가 또는 앱 설치
- 아이폰 사파리: 공유 → 홈 화면에 추가

## 3) 이번 버전의 모바일 변경점
- 하단 메뉴는 항상 보이지 않고, 필요할 때만 `☰ 메뉴` 버튼으로 열림
- 아이콘과 텍스트는 다시 크게 조정
- 첫 화면, 카드, 버튼, 플랜 구성이 모바일 우선으로 재정렬
- 버튼 눌림 애니메이션 추가

## 4) APK 관련
이 압축파일 안에 바로 설치되는 APK 파일이 포함된 것은 아닙니다.
현재 구조는 PWA(웹앱) 형태이며, 홈 화면에 추가하면 앱처럼 사용할 수 있습니다.
추후 APK가 필요하면 PWABuilder 또는 Android Studio 래핑 방식으로 변환할 수 있습니다.


## 모바일 푸시 사용 안내 (실푸시)
- Android: Chrome에서 접속 → 로그인 → "알림 설정" 클릭 → 알림 허용
- iPhone: Safari에서 접속 → 공유 → 홈 화면에 추가 → 홈 화면 앱으로 실행 → 로그인 → "알림 설정" 클릭 → 알림 허용
- iPhone은 일반 사파리 탭보다 홈 화면 앱 상태에서 푸시가 훨씬 안정적입니다.
- 서버는 Web Push 표준(VAPID + Service Worker)을 사용합니다. HTTPS 배포가 반드시 필요합니다.


## 2026-03 추가 확인 사항
- `192.168.x.x`, `10.x.x.x`, `172.16~31.x.x` 같은 내부 IP 주소에서는 모바일 푸시가 정상 동작하지 않을 수 있습니다.
- 테스트는 반드시 **Render 주소 또는 연결된 정식 HTTPS 도메인**에서 진행하세요.
- 내부망 주소에서는 권한이 차단처럼 보일 수 있지만, 실제로는 브라우저 보안 정책 때문에 푸시 자체가 제한되는 경우가 많습니다.
